## Architecture — Read This First

### Build Pipeline
```
npm run build
  → scripts/build.js          (main build orchestrator)
  → scripts/generate-ar-index.js  (generates public/ar/index.html from public/index.html)
  → scripts/verify-dist.js    (fails build if dist is stale or Arabic H1 is English)
  → scripts/version.js        (syncs version across all files)
  → dist/                     (output — what Cloudflare serves)
```

**NEVER edit `public/ar/index.html` directly** — it is generated by `generate-ar-index.js` from `public/index.html`. All changes to Arabic page must go through:
1. `public/index.html` (structure)
2. `public/js/i18n.js` (translations)
3. `scripts/generate-ar-index.js` (generation logic)

### Deployment
```
Edit source files → npm run build → git commit → git push → Cloudflare auto-deploys
```
The `predeploy` script in `package.json` runs `verify-dist.js` first — build fails if checks don't pass.

### CSS Layers (3 files, load in this order)
1. `public/css/app.css` — compiled PostCSS bundle, main design system
   - Defines: `--bg`, `--surface`, `--surface-border`, `--text`, `--text-muted`, `--accent`, `--success`, `--error`, `--warn`
   - Dark mode via `html.dark` class
2. `public/css/tokens.css` — named scale tokens on top of app.css
   - Defines: `--radius-*`, `--text-*`, `--space-*`, `--color-*`, `--btn-*`, `--card-*`
   - Has `html.dark` overrides for semantic colours
3. `public/css/nexus-extras.css` — tool-specific CSS, uses tokens from above
   - Add all new tool CSS here, use `var(--token-name)` not hardcoded values

**`public/css/styles.css`** — separate design system for guide/docs pages only. NOT loaded by main app.

### JavaScript Architecture
- All tool JS is in `public/js/tools/` as self-contained IIFEs
- Shared utilities: `window.NexusTools` (toast, downloadBlob, formatBytes, bindDropZone, etc.)
- i18n: `window.__NEXUS_TF(key, vars)` for translations, `window.__NEXUS_I18N` for raw strings
- Tool routing: `window.__NEXUS_NAVIGATE_TOOL(toolId)` to switch tabs
- Compression: `window.__NEXUS_APPLY_COMPRESSION_VALUES(preset)`, `window.__NEXUS_COMPRESS_ADD_FILES(files)`
- Goal picker: `public/js/goal-picker.js` — checks `nexus-onboarded` localStorage key

## i18n System

**File:** `public/js/i18n.js` (~1500 lines)
**Pattern:** `window.__NEXUS_I18N = { en: { key: 'value' }, ar: { key: 'قيمة' } }`
**Usage in HTML:** `data-i18n="keyName"` on any element
**Usage in JS:** `window.__NEXUS_TF('keyName', { var: value }, 'fallback')`

**Rule:** Every new UI string MUST have both `en` and `ar` keys. Never hardcode English text in HTML that should be translatable.

**Arabic generation:** `scripts/generate-ar-index.js` applies all `data-i18n` keys to the Arabic page automatically. Build will throw if English hero text leaks into Arabic output.

## Design System — Key Rules

### Always Use Tokens
```css
/* CORRECT */
border-radius: var(--radius-lg, 12px);
font-size: var(--text-sm, 0.75rem);
color: var(--color-success, #22c55e);

/* WRONG — hardcoded values */
border-radius: 12px;
font-size: 0.75rem;
color: #22c55e;
```

### CSS Variable Reference
```
--bg              page background
--surface         card/panel background
--surface-border  borders (alias: --border)
--text            primary text
--text-muted      secondary text (alias: --muted)
--accent          brand green (#9aa8e8 dark / #4a5fc1 light)
--success         success green
--error           error red
--warn            warning amber

--radius-xs/sm/md/lg/xl/pill   border radius scale
--text-xs/sm/ui/base/md/lg/xl/2xl/3xl   font size scale
--space-1 through --space-12   spacing scale
--color-success/error/warn/info/ai   semantic colours (dark mode aware)
```

### Button Classes
- `.btn-primary` — green pill, main CTA (colour: `--accent`, text: `#000` dark / `#fff` light)
- `.btn-secondary` — outlined pill, secondary actions
- `.btn-ghost` — no border, icon buttons and subtle actions
- `.btn-danger` — red outlined, destructive actions (clear, delete)
- `.btn-print` — subtle, print actions

### Layout Classes
- `.tool-layout` — 2-col grid (1fr + 300px sidebar)
- `.tool-layout-wide` — 2-col grid (1fr + 340px sidebar)
- `.tool-layout-single` — single column, max 680px
- Collapses to 1 column at 760px breakpoint

## RTL / Arabic Rules

- Arabic page uses `<html lang="ar" dir="rtl">`
- All layout must work RTL — use `margin-inline-start` not `margin-left`
- `app.css` has RTL overrides for common patterns
- `nexus-extras.css` has `[dir=rtl]` overrides for drop-zone, tool-tabs, etc.
- Test every new CSS change in RTL — sidebar should flip to left, text aligns right

- Savings badge: `.result-savings-badge` on result cards in `app.js` — prominent `−{n}%` shown on completion; `is-minimal` class when < 5%

## AI Tools — Special Notes

### AI Upscaler
- ONNX model self-hosted at `public/models/realesrgan-x4.onnx` (~4.87 MB)
- Worker: `public/js/upscaler-worker.mjs`
- Model loaded lazily on tab open, cached via Cache API key `nexus-esrgan-model-v2`

### Photo Checker (face-api.js)
- Models loaded from jsDelivr CDN lazily on tab open
- Uses TinyFaceDetector + face_landmark_68_tiny
- 10 checks: face detected, face size 55-85%, centred, eyes open, head tilt <5°, JPEG format, file size, dimensions, white background, no glasses

## Common Gotchas

1. **Never edit `public/ar/index.html` directly** — always regenerated by build
2. **Never run `npx wrangler deploy` directly** — use `git push`, Cloudflare auto-deploys
3. **Always run `npm run build` before committing** — verify-dist.js will catch stale files
4. **New i18n keys** — must be in BOTH `en` AND `ar` objects or build may warn
5. **`tabUpscaler` key** — do NOT include emoji prefix (🔬 is already in the HTML icon span)
6. **CSS colours** — never hardcode `#22c55e`, use `var(--color-success)` etc.
7. **`margin-left/right`** — use `margin-inline-start/end` for RTL compatibility
8. **Global transitions** — the narrowed rule in nexus-extras.css excludes buttons/inputs on purpose

## What NOT to Do

- Do NOT add inline styles to HTML — use CSS classes only
- Do NOT use `localStorage` for anything user-facing beyond preferences
- Do NOT upload files to any server — all processing must be browser-local
- Do NOT add new npm dependencies without checking bundle size impact
- Do NOT break the Arabic RTL layout when adding new CSS
- Do NOT use `window.alert()` — use `window.NexusTools.toast()` instead
- Do NOT hardcode version strings — use `scripts/version.js`
